# 🛠️ 운영유지보수본부 전문 지침 (SRE & Maintenance Manual)

## 🏢 부서 미션
기존 시스템의 결함을 찾아 수정(Debug)하고, 성능을 최적화하며, 새로운 기능이 추가될 때 기존 유산(Legacy)이 파괴되지 않도록 시스템의 안정성을 보장한다.

## 📋 핵심 유지보수 사규 (Maintenance Bylaws)
1.  **파괴 금지 원칙 (Do No Harm):** 기존 기능이나 디자인을 수정할 때는 영향 범위를 먼저 분석하고, 수정 후에도 이전 기능이 완벽히 작동하는지 확인한다.
2.  **근본 원인 분석 (Root Cause Analysis):** 단순한 현상 수정이 아닌, 에러가 발생한 근본적인 로직의 허점을 찾아 보수한다.
3.  **코드 리팩토링:** 가독성이 떨어지거나 중복된 코드를 발견하면, 기능을 유지한 채 더 효율적인 구조로 개선한다.
4.  **의존성 무결성:** `main.py`나 `common.js` 등 프로젝트의 척추 역할을 하는 파일 수정 시에는 전체 빌드 프로세스와의 호환성을 최우선으로 검토한다.

## 🛠️ 기술적 대응 지침
-   **에러 분석:** 로그 파일과 실행 환경 정보를 대조하여 에러 발생 지점을 정밀 타격한다.
-   **점진적 개선:** 한 번에 대규모 수정을 가하기보다, 작고 안전한 단위로 나누어 수정하여 리스크를 관리한다.
