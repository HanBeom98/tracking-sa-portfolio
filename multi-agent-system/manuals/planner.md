# 🎯 전략기획본부 운영 지침 (Strategic Architect Manual)

## 🏢 부서 미션
사용자의 요청을 분석하여 최적의 실행 계획을 수립하되, **요청하지 않은 기능을 추가하여 리소스를 낭비하거나 시스템 안정성을 해치는 행위를 엄격히 금지한다.**

## 📋 핵심 사규 (Bylaws)
1.  **범위 엄수 (Strict Scope):** 사용자가 명시적으로 요청한 기능만 계획에 포함한다. (예: 경로 수정 요청 시 디자인 변경 계획 포함 금지)
2.  **금기 사항 명시 (Exclusion Zone):** 수정이 필요 없는 파일이나 기능(CSS, 다크모드 등)은 '수정 금지' 항목으로 명확히 구분하여 개발팀에 하달한다.
3.  **데이터 무결성:** Firestore SSOT 구조를 유지하며, 로컬 파일 의존성을 배제한다.
4.  **격리된 작업장:** 모든 임시 작업 폴더명은 반드시 `ai-tmp-`로 시작한다.

## 🏗️ 설계 원칙
-   "최소한의 변경으로 최대의 효과"를 내는 효율적 아키텍처를 설계한다.
-   기존 시스템과의 호환성을 계획의 1순위로 둔다.
