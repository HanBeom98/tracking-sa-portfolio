# Project Blueprint: AdSense Optimization for TrackingSA

## Overview
This document outlines the current state of the TrackingSA website and a plan to optimize it for Google AdSense approval. The website is a static site generated by a Python script (`main.py`), which fetches AI news, generates markdown content, converts it to HTML, and deploys it. It currently includes Firebase integration, basic styling, and dark mode functionality.

## Current State Analysis (as of 2026-02-02)

### Website Structure & Content Generation
- **Entry Point:** `index.html` serves as the main page, dynamically populated with news articles.
- **Styling:** `style.css` provides a clean, modern, and responsive design with dark mode support.
- **Client-side Logic:**
    - `common.js`: Handles header injection (`layout/header.html`) and theme toggling.
    - `main.js`: Dedicated to the "animal face test" functionality on `animal_face_test.html`.
- **Server-side Generation (`main.py`):**
    - Fetches AI news via RSS (TechCrunch).
    - Generates markdown article content using the Gemini API based on news titles/summaries.
    - Saves markdown files in `posts/` and converts them to HTML, saving them in `public/`.
    - Generates `public/index.html` with links to articles.
    - Copies static assets to `public/`.
    - Integrates AdSense script and meta tags into all generated HTML pages.
    - Manages article uniqueness and Git commits.
- **Content Quality:** Sample AI-generated articles (e.g., `2026-02-02-강아지를-잃어버렸다면-링ring-비非카메라-사용자에게도-수색대search-party-기능-전면-개방.md`) are well-structured, informative, and include unique "monetization ideas," adding value.

### AdSense Readiness
- **AdSense Integration:** The necessary AdSense script and meta tags are correctly injected into `<head>` sections by `main.py`.
- **Content Quality:** Generally good, appears original and informative.
- **User Experience:** Basic navigation and good visual design.
- **Essential Pages:** "About Us", "Contact Us", and "Privacy Policy" pages have been added and linked in navigation and footers.
- **Duplicate `h1` tags:** The issue of duplicate `h1` tags on article pages has been resolved for better semantic structure.
- **Potential "Thin Content" Risk:** Pages like `inquiry.html`, `edit.html`, `write.html`, and `post.html` may still be considered "thin content" by AdSense if they lack substantial, valuable information or are publicly accessible administrative interfaces.

## Plan for AdSense Optimization

The goal is to ensure the website fully complies with Google AdSense policies, particularly regarding content quality, user experience, and transparency.

### Completed Subtasks:
1.  **Created 'About Us', 'Contact Us', and 'Privacy Policy' placeholder HTML files.**
    - These files contain basic, yet compliant, information and provide a foundation for further content development. They are copied to the `public/` directory by `main.py`.
2.  **Updated `layout/header.html` to include links to the new pages.**
    - Ensures these important pages are easily accessible from the main navigation.
3.  **Updated `main.py` and `index.html` to include links to the new pages in the footer.**
    - Provides consistent access to these pages across the site.
4.  **Modified `main.py` and `style.css` to prevent duplicate `h1` tags in generated article HTML.**
    - Improves SEO and semantic structure by ensuring only one main `<h1>` per article, which is generated from the markdown content itself.

### Next Steps (for user consideration):
1.  **Review and address "Thin Content" pages:**
    *   **`inquiry.html`:** Enhance this page with substantial content explaining the partnership inquiry process, privacy considerations, etc., to avoid being flagged as thin content.
    *   **`edit.html`, `write.html`, `post.html`:** These likely serve administrative or content creation purposes. They should ideally be protected by authentication. If they must be publicly accessible, they should either be excluded from search engines (`<meta name="robots" content="noindex">`) or contain valuable content to avoid AdSense policy violations. They should certainly not display AdSense ads if they are not meant for public consumption.
2.  **Strategic Ad Unit Placement:** Once AdSense approval is granted, strategically integrate actual AdSense ad units (`<ins>` tags) into `main.py`'s HTML generation functions (`generate_article_html`, `generate_index_html`). Ensure placements are user-friendly, do not disrupt content flow, and comply with AdSense ad placement policies (e.g., no excessive ads, clear separation from content).
