# Change Log

## Version 2.1.0 (2026-03-01)
### ✨ New Features & Enhancements
- **내전 K/D 기록 기능 추가 (Sudden Attack)**:
  - **Data Model**: `sa_crew_members` 컬렉션에 `crewKills`, `crewDeaths` 필드를 추가하여, 내전(크루 매치)의 킬/데스 기록을 누적 저장하도록 확장했습니다.
  - **UI/UX**: '우리 크루 내전 기록 분석' 섹션을 4단 그리드로 개편하여, `내전 MMR`, `내전 K/D`, `내전 승률`, `크루 내 위상`을 명확하게 표시하도록 개선했습니다.
- **빌드 시스템 안정성 강화**:
  - `builder.py`의 파일 복사 로직을 개선하여, `src`의 파일 변경 시 항상 최신 버전이 `public` 폴더에 반영되도록 수정했습니다. 이전 빌드 결과물이 덮어씌워지지 않아 발생하던 배포 문제를 원천적으로 해결했습니다.

### 🐛 Bug Fixes
- **시즌 초기화 안정성 강화**:
  - **UI 동기화**: '시즌 초기화' 버튼 클릭 시, 데이터베이스 초기화 후 페이지가 자동으로 새로고침되지 않아 UI에 이전 데이터가 계속 표시되던 문제를 해결했습니다. (`location.reload(true)`)
  - **데이터 무결성**: 초기화 시 `mmr` 값이 `NaN`으로 표시되던 심각한 버그를 해결했습니다. 데이터베이스 쓰기 요청이 보안 규칙에 의해 소리 없이 거부되던 것이 원인이었으며, `firestore.rules`를 수정하여 관리자가 관련 필드를 정상적으로 업데이트할 수 있도록 조치했습니다.
- **환경 변수 주입 시스템 구축**:
  - **하드코딩 제거**: `NEXON_API_KEY`를 소스 코드에서 분리하고, GitHub Actions Secret에서 받아와 빌드 시점에 안전하게 주입하는 시스템을 구축했습니다.
